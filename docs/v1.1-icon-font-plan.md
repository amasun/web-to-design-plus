# v1.1 核心版本计划：究极矢量还原 (Global SVG Sanitizer & Icon Font Parser)

## 核心问题分析
**痛点**：目前官方的 `capture.js` 引擎在抓取网页到 Figma 时，对绝大部分现代图标的兼容性极差。主要表现在：
1. 缺乏本地字体的 **Icon Font** (FontAwesome 等) 会变成方块或问号丢失。
2. **CSS 背景图** (`background-image: url("*.svg")`) 会被处理成无法改变颜色和路径的普通死位图。
3. **图片标签** (`<img src="*.svg">`) 也会被处理成位图。
4. **CSS 遮罩图标** (`mask-image`) 完全不支持，直接变成纯色方块。
5. **内部/外部雪碧图引用** (`<use href="...">`) 往往因为跨域或上下文丢失而抓取不到。
**限制条件**：`capture.js` 是核心引擎，为了保证未来官方版本的兼容性，我们不能也不应该修改其内部逻辑。

## 解决方案：DOM 预处理“偷天换日”法 (Pre-Capture DOM Mutation)
既然不能改 `capture.js`，我们就**在它运行前的一瞬间，启动一个全局的 SVG 净化器 (Global SVG Sanitizer)**。
`capture.js` 天生完美支持最标准的内联 `<svg><path></svg>` 标签。所以我们只要在抓取前，将以上所有 5 种不兼容的 SVG 调用方式以及 Icon Font，**瞬间强制替换（或展开）为纯正的内联 `<svg>`**，抓取完成后再无缝还原。

## 核心架构与应用场景
这个引擎一旦实现，将同时赋能我们插件的两个核心功能：
1. **Web Capture 全局预处理**：彻底解决 HTML 抓取到 Figma 后的图标丢失和丧失矢量可编辑性的问题，实现**秒杀同类竞品**的“究极矢量保真度”。
2. **SVG Grabber 面板资产扩充**：用户打开提取面板时，不仅能看到原来就能抓取的 SVG，还能直接抓取网页里的 Icon Font、mask-image 等复杂图标，并支持一键下载或推送到 Figma。

### 核心实施步骤 (Implementation Steps)

#### 1. 拦截抓取指令 (Intercept Capture Trigger)
在 `inpage-toolbar.js` 中，当用户点击“Entire Screen”或“Select Element”时，**暂停**立即调用 `figma.capture()`。首先进入异步的“预处理等待期”。

#### 2. 全局 SVG 净化器与字体解析 (Global Sanitization & Sniffing)
* **雪碧图展开**：将在 SVG Grabber 中写好的异步加载逻辑复用，把页面里所有的 `<use>`（外链和内链）原地展开成完整的内联 `<svg>`。
* **背景与图片替换**：把 `<img src=".svg">` 和 `background-image` 的 SVG 下载下来，偷偷替换成绝对定位且尺寸一致的内联 `<svg>`。
* **遮罩还原**：提取 `mask-image`，生成对应的 `<svg>`，并将原元素的背景颜色作为 `fill` 赋予它。
* **Icon Font 矢量化**：遍历带有 Icon Font 的节点（包括难搞的伪元素 `::before`），使用 `opentype.js` 下载字体并解析出对应的数学矢量路径（SVG Path）。

#### 3. DOM 临时置换 (DOM Substitution)
由于我们不能直接破坏原始排版，我们需要做极其精密的外科手术：
1. **隐藏原图标**：给包含伪元素或原背景的节点动态注入特殊的 class，配合临时 `<style>` 将其强制隐藏（或透明）。
2. **注入真实的 SVG**：在同一个节点内部，动态创建一个 `<svg viewBox="..."> <path d="..."/></svg>` 标签。
3. **尺寸与颜色对齐**：通过 `getComputedStyle` 读取原图标的 `font-size`、`color` 或盒子大小，精确赋值给临时 `<svg>`，确保其在网页上的视觉占位与原版一模一样。

#### 4. 执行抓取引擎 (Execute capture.js)
此时，网页上的所有复杂图标已经在物理层面上变成了最标准的 `<svg>`。我们调用官方的 `figma.capture()` 引擎。它会将这些临时注入的 SVG 完美转化为 Figma 中的 Vector 图层！

#### 5. 无痕还原 (Post-Capture Revert)
抓取引擎执行完毕（并生成了 JSON 给剪贴板后），我们立即执行清理，移除所有的临时 `<svg>` 节点和隐藏样式，网页瞬间恢复原样。

## 结论与评估
**完全可行，甚至是一场降维打击！**
这个方案把所有的兼容性脏活累活都在“预渲染”阶段解决，交到 Figma 引擎手里的都是完美干净的数据。如果您准备好启动，我们将分为两个阶段：
1. **Phase 1**：集成全局 SVG 净化器（处理 `<use>`、`<img>`、背景图、`mask`）。
2. **Phase 2**：引入 `opentype.js`，攻克最后的堡垒：Icon Font 矢量化转换。
